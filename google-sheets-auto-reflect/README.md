# 3施設別在庫・報告数字 自動反映

紙データから入力した「個体識別番号 + 部位 + 左右 + 重量kg」を、部位ごとの振り分けルールで3施設別に自動集計するGoogleスプレッドシート用Apps Scriptです。

## 作成されるシート

- `入荷入力`: 紙データから1回だけ入力するシート。
- `振り分けルール`: 部位名ごとの反映先施設を設定するシート。
- `3施設別在庫`: 日付・施設・部位別の集計結果。
- `3施設別報告表`: `資料/カンゲンファーム在庫表作成シート.xlsx` のブロック肉在庫フォームに合わせた報告表。
- `前日在庫`: 報告表の `前日` 欄に入れる施設・部位別の重量。
- `設定`: 報告対象日、PDF保存先フォルダIDなど。
- `部位マスタ`: 入力プルダウン用のマスタ。通常は非表示。

## 新しいシートへ手動で導入する場合

1. Googleスプレッドシートを新規作成する。
2. `拡張機能 > Apps Script` を開く。
3. `Code.gs` にこのフォルダの `Code.gs` を貼り付ける。
4. プロジェクト設定の `appsscript.json` にこのフォルダの `appsscript.json` を貼り付ける。
5. スプレッドシートを再読み込みする。
6. メニュー `3施設 自動反映 > 初期セットアップ` を実行する。

初回実行時はGoogleの権限確認が出ます。PDF出力を使うため、Driveへのアクセス許可が必要です。

## 使い方

1. `振り分けルール` で、部位ごとの反映先施設を確認・修正する。
2. 必要に応じて `前日在庫` に、3施設・部位別の前日重量kgを入力する。
3. `入荷入力` に、日付、個体識別番号、部位名、左右、重量kgを入力する。
4. メニュー `3施設 自動反映 > 自動反映` を実行する。
5. `3施設別在庫` と `3施設別報告表` を確認する。
   `3施設別報告表` は、既存Excelの `2026.4.30(3)` と同じく、各施設に `前日 / IN / OUT / 残り` を表示します。
6. `設定` の `報告対象日` を変えた場合は、`日付別報告作成` を実行する。
7. PDFが必要な場合は、`PDF出力` を実行する。

## ボタン化

スプレッドシート上で図形を挿入し、右上メニューから「スクリプトを割り当て」を選ぶと、以下の関数をボタンにできます。

- `autoReflect`
- `createDailyReport`
- `exportReportPdf`

## 初期仕様

- 初期状態では全ての部位の反映先を `今帰仁冷凍施設` にしています。実運用前に `振り分けルール` を必ず修正してください。
- OCRは含めていません。紙データからの初回入力は人が行います。
- 同じ明細を残したまま `自動反映` を再実行しても、`3施設別在庫` は毎回再集計で作り直すため二重計上しません。
- `入荷入力` の `反映先施設`、`反映状況`、`エラー` は自動更新欄です。
- `OUT` は現時点では0で出力します。出庫・移動も自動化する場合は、入力台帳に入出庫区分を追加して拡張します。
